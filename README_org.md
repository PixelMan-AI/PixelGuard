# 🛡️ PixelGuard — پلتفرم هوشمند تشخیص فیشینگ پیامک

> **نسخه:** 3.7 | **مدل زبانی:** ParsBERT | **محیط اجرا:** Google Colab + Google Drive

سامانه PixelGuard یک پلتفرم امنیت سایبری مبتنی بر هوش مصنوعی است که پیامک‌های دریافتی را در سه دسته **ایمن**، **تبلیغاتی** و **مشکوک (فیشینگ)** طبقه‌بندی می‌کند. این سیستم ترکیبی از مدل زبانی فارسی ParsBERT و موتور قوانین مبتنی بر Regex است و از طریق یک رابط کاربری گرافیکی تحت وب در دسترس است.

## 🧠 معماری سیستم

PixelGuard از یک **موتور هیبریدی چهار لایه** برای تحلیل پیامک استفاده می‌کند:

```

        │
        ▼

│  لایه اول: پیش‌پردازش هوشمند        │

│  • نرمال‌سازی حروف عربی/فارسی      │
│  • استخراج دامنه URL               │
└──────────────┬──────────────────────┘
               │
               ▼
┌───────────────────────────────────────┐
│  لایه دوم: موتور قوانین (Rule-Based) │
│  • تطبیق با الگوهای Regex فیشینگ    │
│  • بررسی دامنه‌های مشکوک             │
│  • اعتبارسنجی سرشماره رسمی          │
└──────────────┬────────────────────────┘
               │
               ▼
```

## 🧪 نحوه تست محصول

### روش ۱: تست دستی از طریق رابط کاربری

۱. لینک Serveo را در مرورگر باز کنید
۲. در فیلد **«فرستنده شناسایی شده»** سرشماره را وارد کنید (مثال: `09121234567` یا `adliran`)
۳. در فیلد **«متن کامل پیامک»** محتوای پیامک را بنویسید
۴. دکمه **«🔍 تحلیل پیامک»** را بزنید

## ⚠️ نکات مهم و محدودیت‌ها

**درباره تونل Serveo:**
- لینک تونل پس از هر بار اجرای مجدد کولب تغییر می‌کند
- اگر تونل قطع شد، فقط سل مربوط به `app_online.py` را دوباره اجرا کنید
- در صورت قطع بودن `serveo.net` می‌توانید از `ngrok` به عنوان جایگزین استفاده کنید

**درباره مدل:**
- مدل برای **متون فارسی** بهینه‌سازی شده؛ عملکرد روی پیامک‌های انگلیسی ممکن است ضعیف‌تر باشد
- حداکثر طول متن ورودی: **128 توکن** (متون بلندتر کوتاه می‌شوند)
- لایه کش شبکه (`lru_cache`) تا **256 لینک کوتاه** را در رم ذخیره می‌کند

## 📊 شاخص‌های ارزیابی مدل

| متریک | توضیح |
|---|---|
| **Overall Accuracy** | دقت کلی تشخیص هر ۳ کلاس |
| **Recall (Sensitivity)** | نرخ شکار فیشینگ‌های واقعی (TP / (TP + FN)) |
| **Specificity** | نرخ صحت تایید پیامک‌های ایمن (TN / (TN + FP)) |
| **F1-Macro** | میانگین F1 برای کلاس‌های نامتوازن |
| **Throughput (RPS)** | تعداد درخواست‌های پردازش‌شده در ثانیه |
| **Avg Latency** | میانگین زمان پاسخ‌دهی سرور |
